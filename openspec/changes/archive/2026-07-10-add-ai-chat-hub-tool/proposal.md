# 变更提案：AI Chat Hub

## 概述

添加一个 AI 并发聊天管理工具，允许用户在统一界面中同时与多个 AI 提供商（ChatGPT、Gemini、DeepSeek、Grok 等）进行对话，通过单一输入框向所有选中的 AI 发送相同问题，并对比它们的回答。

## 能力描述

**核心能力：** 多 AI 并发对话管理

用户可以：
1. 选择要使用的 AI 提供商（支持 ChatGPT、Gemini、DeepSeek、Grok 等）
2. 在统一的输入框中输入问题
3. 同时向所有选中的 AI 发送请求
4. 在分屏或平铺布局中查看各个 AI 的实时响应
5. 对比不同 AI 的回答质量和风格
6. 在浏览器本地配置和管理各个 AI 的 API 密钥

## 目标用户

**主要用户：** 开发者进行技术调研

**使用场景：**
- 需要快速获取多个 AI 对技术问题的不同见解
- 评估不同 AI 模型对特定问题的理解和回答能力
- 对比多个 AI 的响应速度和答案质量

## 设计说明

### 架构模式

- **模式：** 纯前端（client-only）
- **API 调用：** 直接从浏览器调用各 AI 提供商的公开 API
- **密钥管理：** 使用 localStorage 在浏览器本地存储用户配置的 API 密钥
- **无后端依赖：** 不需要 api-gateway 或 service 模块

### UI 布局

**布局模式：** 支持两种视图模式切换

1. **平铺模式（Grid）：** 所有 AI 响应区域同时可见，网格布局
2. **Tab 模式：** 使用选项卡切换查看不同 AI 的响应

**核心组件：**
- AI 提供商选择器（多选）
- 统一输入框（支持多行文本）
- 视图模式切换器（Grid/Tab）
- AI 响应区域（支持流式显示）
- API 密钥配置面板

### 技术实现

- **AI 集成：** 使用各 AI 提供商的官方 SDK 或 REST API
- **流式响应：** 支持 Server-Sent Events (SSE) 或 streaming API
- **状态管理：** React hooks 管理 AI 选择、输入内容、响应状态
- **本地存储：** localStorage 存储 API 密钥和用户偏好设置

## 验收标准

### 功能验收

- [ ] 用户可以选择至少 4 个 AI 提供商（ChatGPT、Gemini、DeepSeek、Grok）
- [ ] 用户可以在统一输入框中输入问题并发送给所有选中的 AI
- [ ] 支持平铺和 Tab 两种视图模式切换
- [ ] 各 AI 的响应支持流式显示（实时更新）
- [ ] 用户可以配置和管理各 AI 的 API 密钥
- [ ] API 密钥安全存储在浏览器本地，不上传到服务器

### 质量验收

- [ ] 所有 UI 文本支持中英文国际化
- [ ] 响应式布局，适配桌面和平板设备
- [ ] 错误处理：API 调用失败时显示友好错误信息
- [ ] 加载状态：显示各 AI 的请求状态（加载中、成功、失败）
- [ ] 单元测试覆盖核心逻辑（API 调用、状态管理）

### 性能验收

- [ ] 多个 AI 并发请求不阻塞 UI
- [ ] 流式响应延迟 < 100ms
- [ ] 首次加载时间 < 2s

## 非功能需求

- **安全性：** API 密钥仅存储在用户浏览器本地，不传输到后端
- **可扩展性：** 架构支持未来添加更多 AI 提供商
- **用户体验：** 清晰的视觉反馈，区分不同 AI 的响应区域
- **错误恢复：** 单个 AI 请求失败不影响其他 AI 的响应

## 依赖和风险

### 依赖

- 各 AI 提供商的公开 API 和 SDK
- 用户需要自行获取各 AI 的 API 密钥

### 风险

- **API 变更：** AI 提供商可能更改 API 接口或认证方式
- **CORS 限制：** 某些 AI API 可能不支持浏览器直接调用，需要代理
- **速率限制：** 用户的 API 密钥可能受到速率限制

### 缓解措施

- 使用官方 SDK 降低 API 变更影响
- 提供清晰的文档说明如何获取和配置 API 密钥
- 在 UI 中显示速率限制错误并提供重试机制
