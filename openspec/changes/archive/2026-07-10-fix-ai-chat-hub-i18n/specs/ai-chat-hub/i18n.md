# AI Chat Hub - 国际化能力规范

## 概述

AI Chat Hub 工具支持中英文双语界面，所有 UI 文本通过 react-i18next 进行国际化。

## 需求

### REQ-I18N-001: 统一 Namespace 配置
**状态**: MODIFIED  
**优先级**: P0

**描述**:
所有组件必须使用统一的 i18n namespace `toolAiChatHub` 来加载翻译资源。

**场景**:
```typescript
// 正确用法
import { useTranslation } from 'react-i18next'

const MyComponent = () => {
  const { t } = useTranslation('toolAiChatHub')
  return <div>{t('messages.selectAI')}</div>
}
```

**验收标准**:
- 所有组件使用 `useTranslation('toolAiChatHub')`
- 翻译文本正确显示
- 语言切换功能正常

**修改原因**:
修复了子组件未指定 namespace 导致的翻译加载失败问题。

### REQ-I18N-003: Manifest 配置
**状态**: UNCHANGED  
**优先级**: P0

**描述**:
Tool manifest 正确配置 namespace 和翻译文件加载。

**配置**:
```typescript
const toolAiChatHubManifest = defineToolManifest({
  id: 'tool-ai-chat-hub',
  namespace: 'toolAiChatHub',
  loadMessages: {
    zh: () => import('./src/locales/zh.json'),
    en: () => import('./src/locales/en.json'),
  },
  // ...
})
```

**验收标准**:
- namespace 设置为 `toolAiChatHub`
- 正确加载中英文翻译文件
- 翻译文件路径正确

### REQ-I18N-004: 测试覆盖
**状态**: MODIFIED  
**优先级**: P1

**描述**:
所有组件测试必须正确配置 i18n mock，使用正确的 namespace。

**测试配置**:
```typescript
import i18n from 'i18next'
import { I18nextProvider } from 'react-i18next'

i18n.init({
  lng: 'en',
  fallbackLng: 'en',
  ns: ['toolAiChatHub'],
  defaultNS: 'toolAiChatHub',
  resources: {
    en: {
      toolAiChatHub: {
        // test translations
      }
    }
  }
})

// In test
<I18nextProvider i18n={i18n}>
  <Component />
</I18nextProvider>
```

**验收标准**:
- 所有组件测试使用正确的 namespace
- 测试通过
- 无 i18n 警告

**修改原因**:
确保测试环境与生产环境的 i18n 配置一致。

## 技术约束

1. **Namespace 命名**: 必须使用 `toolAiChatHub`，与 manifest 中的 namespace 一致
2. **翻译文件位置**: `src/locales/zh.json` 和 `src/locales/en.json`
3. **Hook 使用**: 所有组件使用 `useTranslation('toolAiChatHub')`
4. **键值对应**: 中英文翻译文件的 key 必须完全一致

## 依赖

- react-i18next: ^13.x
- @toolbox/tool-registry: 提供 manifest 定义
- @toolbox/i18n-runtime: 工具级 i18n 加载支持
