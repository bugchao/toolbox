# 工具规划约定

约定：所有工具的 `已开发 / 待开发 / 调研中` 状态，统一以 [TOOLS_ROADMAP.md](./TOOLS_ROADMAP.md) 为准。

## 分类命名

后续规划和更新统一使用当前应用中的分类 key：

| 分类 | 说明 |
|------|------|
| `utility` | 图片、文档、格式转换、生成器等实用工具 |
| `dev` | 编解码、调试、格式化、开发辅助 |
| `query` | 基础查询类工具 |
| `news` | 资讯聚合 |
| `ai` | AI 辅助或生成类工具 |
| `network` | DNS / IP / 子网 / DHCP / GSLB / 安全 / 运维 |
| `life` | 习惯、健康、消费、日常效率 |
| `travel` | 行程、预算、翻译、旅行辅助 |
| `learning` | 学习效率、知识理解、编程与语言学习 |

## 新增规划怎么写

新增规划时，只做一件事：编辑 [TOOLS_ROADMAP.md](./TOOLS_ROADMAP.md)。

### 已经确定工具名和路由

直接在“当前待办”表格新增一行，填写：

- 工具名称
- 分类
- 建议路径
- 状态
- 优先级
- 备注

### 只有方向，还没定名字

先写进后续计划区，等工具名称和路径稳定后再挪到“当前待办”。

## 上线后怎么更新

工具开发完成后，按下面顺序更新：

1. 在 [TOOLS_ROADMAP.md](./TOOLS_ROADMAP.md) 的“已开发工具”对应分类补一行。
2. 从“当前待办”里删除或标记该工具。
3. 如果导航、首页或 manifest 发生变化，同步更新对应代码。
4. 如果根目录概览文档有必要提到该方向，再更新 [TOOLS_LIST.md](../TOOLS_LIST.md) 或 [README.md](../README.md) 的概览说明。

## 什么不要再做

- 不要再新建一份平行的“完整工具清单”。
- 不要在活跃目录里放日期快照、批次统计、一次性核对表。
- 不要在根目录文档里维护精确工具数量和逐项工具状态。

这些内容如果必须保留，请放到 `docs/archive/`。
