# 工具规划约定（如何提规划、如何避免重复）

约定：**所有「已开发 / 待开发 / 调研」工具以 [TOOLS_ROADMAP.md](TOOLS_ROADMAP.md) 为唯一来源**。后续给人或给 AI 发规划时，按下面方式操作即可。

---

## 1. 分类目录（规划必须带分类）

规划或新增工具时，**必须**指定以下其一：

| 分类 | 含义 |
|------|------|
| `utils` | 实用工具（图片/文档/表格/格式转换、生成器等） |
| `dev` | 研发工具（编解码、正则、时间戳、哈希、代码格式化等） |
| `query` | 查询工具（邮编、天气、IP、颜色等） |
| `news` | 资讯工具（资讯聚合、热点） |
| `ai` | AI 工具（会议纪要、文案、设计、PPT 等） |
| `network` | 网络与基础设施（DNS/IP/子网/DHCP/IPAM/GSLB/运维/安全诊断等，见 TOOLS_ROADMAP 后续计划） |

---

## 2. 如何立项（从规划到待办）

**立项** = 确定「要做这个工具」并进入可执行清单。两种常见来源：

| 来源 | 操作 |
|------|------|
| **后续计划里的某一项**（如「网络工具集」100 项、或「后续计划（待立项/未命名）」表格） | 1. 定好**工具名称**和**建议路径**（如 `/dns-query`）<br>2. 在 **「三、待开发/调研工具」→「当前待办」表格** 里**新增一行**<br>3. 填齐：工具名称、分类、建议路径、状态、优先级、备注<br>4. 可选：在后续计划列表里删掉或标注该条，避免重复 |
| **全新想法**（不在任何列表里） | 1. 先看「二、已开发」和「三、当前待办」，确认**没有同功能/同名**工具<br>2. 在「当前待办」表格**新增一行**，填齐六列 |

**当前待办表格必填**：

| 列 | 说明 |
|----|------|
| 工具名称 | 对外展示名称，如「DNS 查询工具」 |
| 分类 | 必填，且必须是「一、分类目录」中的其一：`utils` / `dev` / `query` / `news` / `ai` / `network` |
| 建议路径 | 路由，如 `/dns-query`，上线后即固定 |
| 状态 | `调研中` / `待开发` / `开发中`（上线后整行移到「二、已开发」） |
| 优先级 | 如 P0 / P1 / P2，用于排期 |
| 备注 | 需求要点、技术要点、或调研结论 |

立项完成后，开发时以「当前待办」该行为需求来源；上线后按「4. 上线后怎么更新」操作。

---

## 3. 已开发放哪里 / 待开发放哪里

- **已开发**：全部列在 [TOOLS_ROADMAP.md](TOOLS_ROADMAP.md) 的 **「二、已开发工具」**，并写明 **路由路径** 和 **代码位置**（`apps/web/src/pages/xxx.tsx` 或 `tools/tool-xxx`）。  
  → 查「有没有做过」只看这里，避免重复开发。

- **待开发 / 调研**：全部列在 [TOOLS_ROADMAP.md](TOOLS_ROADMAP.md) 的 **「三、待开发/调研工具」** 里。  
  - 已有名称和路径的：写在「当前待办」表格。  
  - 仅有方向、尚未命名的：写在「后续计划（待立项/未命名）」表格，确定后再挪到当前待办。  
  → 新想法、调研中的工具都写在这里，不要单独建零散文档。

---

## 4. 后续规划怎么「发」给协作方 / AI

- **口头/聊天**：可以说「在 TOOLS_ROADMAP 里加一个待开发：xxx，分类 ai，路径 /xxx」。  
- **实际操作**：由人或 AI 直接编辑 **`docs/TOOLS_ROADMAP.md`**：  
  - 在「三、待开发/调研工具」表格**新增一行**；  
  - 填齐：工具名称、分类、建议路径、状态（调研中/待开发/开发中）、优先级、备注。

这样「规划」只改一个文件，已开发/待开发一目了然。

---

## 5. 上线后怎么更新（避免清单和代码不一致）

1. 在 **TOOLS_ROADMAP.md** 的「二、已开发工具」对应分类表里**新增一行**：工具名称、路由路径、代码位置。  
2. 在「三、待开发/调研工具」里**删掉**该工具那一行。  
3. 同步更新：  
   - [TOOLS_LIST.md](../TOOLS_LIST.md)（已上线表格与统计）；  
   - [Layout.tsx](../apps/web/src/components/Layout.tsx)（导航）；  
   - [Home.tsx](../apps/web/src/pages/Home.tsx)（首页卡片）；  
   - [App.tsx](../apps/web/src/App.tsx)（路由）。

---

## 6. 给 AI 的简短指令示例

- 「按 TOOLS_ROADMAP 的待开发列表，优先做 P1」  
- 「在 TOOLS_ROADMAP 里加一个待开发：YYY，分类 utils，路径 /yyy，状态待开发」  
- 「查一下 TOOLS_ROADMAP，看 CSV 编辑器有没有被开发过」  

以上都表示：以 **docs/TOOLS_ROADMAP.md** 为准，已开发看「二」，待开发/规划看「三」，分类按「一」。
